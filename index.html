Procedural Music Format

An open JSON-based format for procedural and AI-assisted music composition data. The recipe behind the music, not the audio itself.

Version 1.0 MIT License JSON
Read the Spec View on GitHub

What PMF stores

Identity

Seed, key, BPM, mood, bar count — everything that makes this composition unique.

Harmony

Chord progression with names and scale degrees, bass pattern, arp structure.

Melody

Motifs in ABC notation with interval and rhythm descriptions — readable by LLMs and humans alike.

Arrangement

Section names, bar-gain masks, and melody-on bars — the full dynamic shape of the loop.

Synthesis

Per-voice parameters with semantic labels alongside raw values. Engine-defined but structured.

Summary

A natural language paragraph generated by the engine. An LLM can read this alone and understand the composition.

Minimal example

{
  "pmf_version": "1.0",
  "engine":      "my-engine",
  "summary":     "Upbeat 4-bar loop in C major at 120 BPM.",
  "identity": {
    "seed": 1,
    "bpm":  120.0,
    "key":  "C major",
    "bars": 4
  },
  "render_key": {
    "sample_rate":         44100,
    "bars":                4,
    "stem_format":         "stereo",
    "loop_mode":           "looped",
    "engine_version_major": 1
  }
}

Why not MIDI, MusicXML, or ABC?

FormatDesigned forWhat's missing
MIDIPerformance captureBinary; no compositional intent
MusicXMLScore printingXML verbosity destroys LLM readability
ABC NotationScore transcriptionNote-level only; no seed, mood, or generative params
ChordProChord sheetsNo melody, arrangement, or synthesis detail

PMF describes how to generate music, not music that already exists. It complements MIDI and WAV — an engine can export all three from the same render.
